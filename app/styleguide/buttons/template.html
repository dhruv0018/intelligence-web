Styles

Corner Radius

The corder radius on all buttons should be kept to a consistent 3px.

Sizing and Padding

Buttons can be one of three sizes: small, medium, and large. Medium is the standard size with 16px font-size.

  • Small - 12px
  • Medium - 16px
  • Large - 20px

Buttons with Icons

Buttons can be accompanied by icons to better describe the action being taken.

Placement

Buttons placed together should have padding between them and the area around them.

Text Links

Text links should have padding consistent with buttons.

Icon Shortcuts

Icon shortcuts should keep even padding between them.

Default Buttons

Every button is default unless it falls into one of the categories below.

Transaction Buttons

Button that indicates a purchase or order flow. This style allows us to consistently alert the user that the selected action will lead to a purchase or ordering process.

  • Order Now
  • Submit for Breakdown

Default Button Example
Button with action areas 1 and 2, optional header, supporting text and supplemental actions.

Important Action Buttons

Button used to initiate processes that are integral for Krossover users, including adding content to the platform. This style allows us to clearly show the user how to initiate these processes. There should only be one of these buttons on any given page and these buttons should not be used in conjunction with a primary action button.

  • Add game
  • Add player
  • Add new user
  • Add new team
  • Add new school

Default Button Example
Button with action areas 1 and 2, optional header, supporting text and supplemental actions.

Primary Action Buttons

Button that represents a positive action that completes a process or flow. This style allows us to both distinguish these options from destructive actions and alert the user that there are no more steps to complete. Most commonly used in confirmation modals, but also for completing inline actions.

  • Save
  • Submit
  • Add
  • Copy
  • Confirm

Default Button Example
Button with action areas 1 and 2, optional header, supporting text and supplemental actions.

Destructive Action Buttons

Limited to those potentially destructive actions which involve removing or deleting. If the action that completes a process falls into this definition, a destructive action should be used instead of a primary action button. This style allows us to draw the user's attention to what could happen when they confirm.

  • Delete
  • Remove
